General

What is the OpenGeo Suite?

The OpenGeo Suite is a complete geospatial software stack, consisting of a spatial database (PostGIS), application server (GeoServer), tile cache server (GeoWebCache), and application client libraries (OpenLayers and GeoExt). It includes all the tools necessary to create a full geospatial solution, including sample applications, documentation, and code examples.

Troubleshooting

How do I start or stop the OpenGeo Suite?

The simplest way to start and stop the OpenGeo Suite is through the Dashboard, by clicking the Start/Stop button on the top right. You can also manually start and stop from the command line by typing "opengeo-suite start" or "opengeo-suite stop" in the root of your application directory.

Return to top

How do I know if the OpenGeo Suite is running?

You can check the status of the OpenGeo Suite in the top right of the Dashboard. Also, many links in the Dashboard are disabled when the OpenGeo Suite is running. If this link is enabled, the OpenGeo Suite is running.

Return to top

Why can't I start the OpenGeo Suite?

The most common reason why the OpenGeo Suite is unable to start is because of a port conflict. By default, the OpenGeo Suite web applications run on port 8080 of the host machine, and if another service using that port, the OpenGeo Suite will be unable to run. To change this port, go to the Preferences page, change the Primary Port, click Save, then restart the OpenGeo Suite. If you are still having trouble starting the service, you may want to check the Logs for other errors. If you are still stuck, please visit the OpenGeo Suite Community Forum and describe your issue and the steps you've taken to resolve it.

If I have a problem, how do I find out the error?

The most helpful place to look is the Logs. Although the logs can be verbose, errors are often displayed in line among other status outputs. You may have to scroll through the logs to find the error in question. Hint: The line that starts with the phrase Caused by can often contain helpful information about the error.

PostGIS

Can I run the OpenGeo Suite if I already have PostGIS installed?

Yes. By default, PostGIS installations run on port 5432, but the PostGIS database included with the OpenGeo Suite runs on port 54321. As long as these port numbers are different, there should be no conflicts. You can change the PostGIS port on the Preferences page.

Return to top

How do I know PostGIS is running?

Since PostGIS is part of the OpenGeo Suite, if the OpenGeo Suite is running then PostGIS will be as well. Check the Dashboard for the status of the OpenGeo Suite or click on the Manage PostGIS link to run PgAdmin, a graphical database management application.

Return to top

What port is PostGIS running on?

By default, PostGIS runs on port 54321, except for Linux packages which use the standard port 5432. This value can be changed on the Preferences page.

Return to top

What is the default username and password for PostGIS?

The default username is "postgres" with no password, except for Linux packages, where the default username and password are both "opengeo". For security reasons, we recommend changing this username and password.

Return to top

How can view/edit my PostGIS database?

The OpenGeo Suite includes PgAdmin, a desktop graphical database management application. Click on the Manage PostGIS link in the Dashboard to run PgAdmin. The command line tool psql is also included.

Return to top

How do I load a shapefile into PostGIS when I don't know its projection?

The OpenGeo Suite User Manual has links to methods of finding the projection, such as utilizing the online service prj2epsg.org. If you are unable to determine the projection, you may still be able to use the Layer Importer and load the shapefiles directly into GeoServer, as GeoServer can often intelligently determine the projection.

Return to top

Can I convert a directory of shapefiles to PostGIS tables?

A graphical method to batch convert shapefiles to PostGIS tables is planned for a future version.

GeoServer

Can I run the OpenGeo Suite if I already have GeoServer installed?

Yes, though GeoServer and the OpenGeo Suite both run on port 8080 by default. If you already have another GeoServer instance running on port 8080, the OpenGeo Suite will not start properly. To change this port, go to the Preferences page, change the Primary Port, click Save, then restart the OpenGeo Suite.

Return to top

Does the GeoServer support connections with ArcSDE or Oracle Spatial databases?

Yes, however there are certain required files that must be manually copied from your own ArcSDE/Oracle installation. The files required for ArcSDE are jsde*.jar and jpe*.jar from your ArcSDE installation. The files required for Oracle Spatial is ojdbc*.jar from your Oracle installation. Copy the required file(s) into the installation directory under webapps/geoserver/WEB-INF/lib from the root of your installation and restart the OpenGeo Suite. You should then see ArcSDE/Oracle as options in the GeoServer list of new data stores.

Return to top

Where is my GeoServer data directory?

Your GeoServer data directory is located inside your user's home directory at .opengeo/data_dir.

Return to top

Why did I get a Java stack trace with GeoServer inside my web browser?

Java stack traces can happen for many reasons, ranging from incorrect requests to software bugs. The first lines of the stack trace, or the line that says Caused by usually has the most helpful information. If you are unable to discern the particular error you're receiving, please create a post on the OpenGeo Suite Community Forum. Please describe what you were doing immediately prior to the error occurring, as well as your operating system, software version, and other relevant information. Enterprise Edition clients can contact OpenGeo support directly.

Return to top

How do I add multiple data sets to GeoServer at once?

The OpenGeo Suite includes a Layer Importer as an extension to GeoServer that allows for batch loading of PostGIS database tables and shapefiles.

OpenLayers

Why is my OpenLayers map blank (or pink)?

Blank or pink maps in OpenLayers are usually caused by either a JavaScript error, or an invalid connection to a web service. A dedicated JavaScript debugger is recommended, such as Firebug for the Firefox browser.

GeoExplorer

Can I edit styles graphically with GeoExplorer?

Yes, GeoExplorer contains a graphical SLD editing toolset.

Return to top

Can I edit vector features or metadata with GeoExplorer?

Yes, GeoExplorer contains full feature editing capabilities.

Return to top

Does GeoExplorer's graphical styling capabilities include all of the functionality in the SLD spec?

GeoExplorer includes the most commonly used tags contained in the SLD 1.0 specification, but does not attain 100% compatibility. Certain extras, such as GeoServer vendor options, are not supported or preserved. It is recommended to make a backup of all SLD files before editing.

Return to top

Why are my vendor options are not rendered when I view an existing SLD in GeoExplorer?

GeoExplorer does not recognize or preserve GeoServer vendor options, which are not part of the SLD specification. It is recommended to make a backup of all SLD files before editing.

Return to top

Why is GeoExplorer blank, with no layers in the layer panel?

This is often caused by a corrupt layer entry in GeoServer, leading to a corrupted WMS capabilities document. You may wish to remove and reload any recent layers or layer groups. The Logs may also have more detailed information.

Return to top

How do I increment the bookmark/map number in GeoExplorer?

The bookmark/map number increments automatically when you compose a new map. It is not editable from inside the application.
